1. ClusterIP (پیش‌فرض)
   
کاربرد: این نوع سرویس فقط دسترسی داخلی درون کلاستر Kubernetes را فراهم می‌کند.

توضیح: زمانی که بخواهید پادها فقط از داخل کلاستر با یکدیگر ارتباط برقرار کنند، از این نوع استفاده می‌شود.

مثال: برنامه‌های داخلی که نیازی به دسترسی خارجی ندارند، مانند پایگاه داده یا سرویس‌های داخلی.

سناریو: شما یک پایگاه داده MySQL دارید که فقط باید توسط یک برنامه وب دیگر درون کلاستر استفاده شود. هیچ کاربر خارجی نباید به این پایگاه داده دسترسی داشته باشد.


فایل YAML برای ClusterIP:
```
apiVersion: v1
kind: Service
metadata:
  name: mysql-service
spec:
  selector:
    app: mysql
  ports:
    - protocol: TCP
      port: 3306
      targetPort: 3306
  type: ClusterIP
```
